---
title: Guards
description: Control de acceso booleano antes de que se ejecute el handler
---

Un `Guard` decide si a una solicitud se le permite continuar. Construye uno con
`gonest.NewGuard(func(guard *gonest.Guard) {...})`.

```go
var AuthGuard = gonest.NewGuard(func(guard *gonest.Guard) {
  guard.Handler(func(req *gonest.Request, res *gonest.Response) bool {
    return req.Header("Authorization") != ""
  })
})
```

- Devolver `false` produce un **403 Forbidden** automático — no se ejecuta ningún Interceptor
  o Handler.
- Provocar un panic con una `Exception` dentro de un guard produce la propia respuesta
  de la exception en lugar de un 403 genérico.
- Los múltiples guards se evalúan en orden de registro y **hacen un cortocircuito (short-circuit) en
  el primer `false`**.

## Aplicando un guard

```go
var UserController = gonest.NewController(func(controller *gonest.Controller) {
  controller.Guards(AuthGuard)
})
```

<Callout type="warn">
  Los Guards son **solo del ámbito del Controller** — no hay `Module.Guards`. También
  **no tienen soporte para `MustInject`** dentro del constructor (builder): un solo guard puede
  adjuntarse a través de muchos controllers/modules, por lo que no hay un solo "propietario" (owner)
  frente al cual resolver dependencias.
</Callout>

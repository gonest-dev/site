---
title: Guards
description: Controle de acesso booleano antes da execução do handler
---

Um `Guard` decide se uma requisição tem permissão para prosseguir. Construa um com
`gonest.NewGuard(func(guard *gonest.Guard) {...})`.

```go
var AuthGuard = gonest.NewGuard(func(guard *gonest.Guard) {
  guard.Handler(func(req *gonest.Request, res *gonest.Response) bool {
    return req.Header("Authorization") != ""
  })
})
```

- Retornar `false` produz um **403 Forbidden** automático — nenhum Interceptor
  ou Handler é executado.
- Dar um panic com uma `Exception` dentro de um guard produz a própria resposta
  da exception em vez de um 403 genérico.
- Múltiplos guards são avaliados na ordem de registro e **encerram a avaliação precocemente (short-circuit) no
  primeiro `false`**.

## Aplicando um guard

```go
var UserController = gonest.NewController(func(controller *gonest.Controller) {
  controller.Guards(AuthGuard)
})
```

<Callout type="warn">
  Os Guards têm escopo **apenas de Controller** — não há `Module.Guards`. Eles
  também **não têm suporte a `MustInject`** dentro do builder: um único guard pode
  ser anexado a vários controllers/modules, de modo que não há um "proprietário" (owner) único
  para resolver dependências a partir dele.
</Callout>

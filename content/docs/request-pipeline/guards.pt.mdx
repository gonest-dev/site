---
title: Guards
description: Controle de acesso booleano antes da execução do handler
---

A `Guard` decides whether a request is allowed to proceed. Build one with
`gonest.NewGuard(func(guard *gonest.Guard) {...})`.

```go
var AuthGuard = gonest.NewGuard(func(guard *gonest.Guard) {
  guard.Handler(func(req *gonest.Request, res *gonest.Response) bool {
    return req.Header("Authorization") != ""
  })
})
```

- Returning `false` produces an automatic **403 Forbidden** — no Interceptor
  or Handler runs.
- Panicking with an `Exception` inside a guard produces that exception's own
  response instead of a generic 403.
- Multiple guards are evaluated in registration order and **short-circuit on
  the first `false`**.

## Applying a guard

```go
var UserController = gonest.NewController(func(controller *gonest.Controller) {
  controller.Guards(AuthGuard)
})
```

<Callout type="warn">
  Guards are **controller-scoped only** — there is no `Module.Guards`. They
  also have **no `MustInject` support** inside the builder: a single guard can
  be attached across many controllers/modules, so there's no single "owner"
  to resolve dependencies against.
</Callout>

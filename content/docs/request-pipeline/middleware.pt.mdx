---
title: Middleware
description: Middleware de requisição no estilo Express
---

O `Middleware` roda antes do roteamento — o mesmo modelo mental que o middleware
do Express. Construa um com `gonest.NewMiddleware(func(middleware *gonest.Middleware) {...})`.

```go
var RequestIdMiddleware = gonest.NewMiddleware(func(middleware *gonest.Middleware) {
  middleware.Handler(func(req *gonest.Request, res *gonest.Response, next gonest.Next) {
    res.SetHeader("X-Request-Id", "...")
    next(req, res)
  })
})
```

Chamar `next(req, res)` continua a cadeia. **Omitir essa chamada encerra (short-circuits)**
a requisição — nada a jusante (downstream) roda.

## Global vs. escopo de controller

```go
var AppModule = gonest.NewModule(func(module *gonest.Module) {
  module.Use(RequestIdMiddleware) // global — root module only
})

var UserController = gonest.NewController(func(controller *gonest.Controller) {
  controller.Use(SomeOtherMiddleware) // controller-scoped
})
```

O middleware global (`Module.Use`, apenas para o Module raiz) sempre roda **antes** do
middleware de nível de Controller (`Controller.Use`).

title: Exceptions & Recuperação de Panic
description: HttpException, exceptions internas e o contrato de recuperação de panic
---

O `gonest.HttpException` é o tipo base que toda exception do gonest embute (embeds).

```go
gonest.NewHttpException().
  SetStatus(http.StatusBadRequest).
  SetName("FooError").
  SetMessage("lorem ipsum").
  SetDetails(details)
```

Qualquer tipo que embuta `HttpException` satisfaz estruturalmente a interface `Exception`
— a camada de recuperação usa uma asserção de tipo (type-assertion), nunca um
type-switch fechado.

## Exceptions embutidas (Built-in exceptions)

| Exception | Status |
|---|---|
| `NotFoundException` | 404 |
| `BadRequestException` | 400 |
| `ConflictException` | 409 |
| `UnauthorizedException` | 401 |
| `ForbiddenException` | 403 |

Cada uma possui um construtor `New*Exception(details any)`.

## Exceptions customizadas

Embuta `gonest.HttpException` exatamente como uma embutida (built-in):

```go
type FooError struct{ gonest.HttpException }

func NewFooError(details any) *FooError {
  return &FooError{
    HttpException: gonest.NewHttpException().
      SetStatus(http.StatusBadRequest).
      SetName("FooError").
      SetMessage("lorem ipsum").
      SetDetails(details),
  }
}
```

## Contrato de Panic → resposta

<Callout type="info">
  Qualquer panic com um valor que satisfaça `Exception` torna-se um JSON
  `{"name": ..., "message": ..., "details": ...}` no próprio código de status daquela
  exception. Qualquer **outro** panic (ponteiro nulo, índice fora dos limites, etc.)
  torna-se um erro 500 genérico com **zero detalhe interno vazado** — o processo
  nunca trava (crashes).
</Callout>

## Constantes de status HTTP

`gonest.HttpStatus*` reexporta todas as 63 constantes padrão `net/http.StatusXxx`
sob o namespace `gonest` (ex: `gonest.HttpStatusNotFound`), para que
o código do handler nunca precise importar `net/http` apenas para códigos de status.

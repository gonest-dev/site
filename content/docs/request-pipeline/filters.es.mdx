---
title: Filters
description: Capturando tipos específicos de exception y sobrescribiendo su respuesta
---

Un `Filter` captura un tipo de exception **específico** y sobrescribe su respuesta
predeterminada. Construye uno con `gonest.NewFilter(func(filter *gonest.Filter) {...})`.

```go
var FooFilter = gonest.NewFilter(func(filter *gonest.Filter) {
  filter.Catch(&FooError{}, func(req *gonest.Request, res *gonest.Response, exc *FooError) {
    res.Status(http.StatusTeapot).Json(map[string]any{"custom": true})
  })
})
```

`Catch(exemplar, handler)` coincide por el **tipo concreto** exacto del
valor de panic (a través de `reflect.TypeOf`) — no por una interfaz o una jerarquía de tipos.
Los tipos no capturados pasan al handler de panic predeterminado `{name, message, details}`
(consulta [Exceptions](/docs/request-pipeline/exceptions)).

## Aplicando un filter

```go
var UserController = gonest.NewController(func(controller *gonest.Controller) {
  controller.Filters(FooFilter)
})

var AppModule = gonest.NewModule(func(module *gonest.Module) {
  module.Filters(FooFilter) // global — root module only
})
```

<Callout type="info">
  Un filter a nivel de Controller le gana a uno global para el mismo tipo capturado.
  `Filter` envuelve (wraps) **todo** el pipeline — Middleware, Guard, Interceptor, y
  Handler — por lo que captura panics desde cualquier lugar dentro de esa cadena, no solo
  desde el Handler. Consulta [Visión General y Orden de Ejecución](/docs/request-pipeline).
</Callout>

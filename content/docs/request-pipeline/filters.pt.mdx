---
title: Filters
description: Capturando tipos específicos de exception e sobrescrevendo sua resposta
---

Um `Filter` captura um tipo **específico** de exception e sobrescreve sua resposta
padrão. Construa um com `gonest.NewFilter(func(filter *gonest.Filter) {...})`.

```go
var FooFilter = gonest.NewFilter(func(filter *gonest.Filter) {
  filter.Catch(&FooError{}, func(req *gonest.Request, res *gonest.Response, exc *FooError) {
    res.Status(http.StatusTeapot).Json(map[string]any{"custom": true})
  })
})
```

`Catch(exemplar, handler)` faz a correspondência pelo **tipo concreto** exato do
valor do panic (via `reflect.TypeOf`) — não por uma interface ou uma hierarquia de tipos.
Tipos não capturados caem para o handler de panic padrão `{name, message, details}`
(veja [Exceptions](/docs/request-pipeline/exceptions)).

## Aplicando um filter

```go
var UserController = gonest.NewController(func(controller *gonest.Controller) {
  controller.Filters(FooFilter)
})

var AppModule = gonest.NewModule(func(module *gonest.Module) {
  module.Filters(FooFilter) // global — root module only
})
```

<Callout type="info">
  Um filter no nível de Controller ganha de um filter global para o mesmo tipo capturado.
  O `Filter` envolve (wraps) **todo** o pipeline — Middleware, Guard, Interceptor e
  Handler — portanto, ele captura panics de qualquer lugar dentro dessa cadeia, não apenas
  do Handler. Veja [Visão Geral & Ordem de Execução](/docs/request-pipeline).
</Callout>

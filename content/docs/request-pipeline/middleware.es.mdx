---
title: Middleware
description: Middleware de solicitudes al estilo Express
---

El `Middleware` se ejecuta antes del enrutamiento — el mismo modelo mental que el middleware
de Express. Construye uno con `gonest.NewMiddleware(func(middleware *gonest.Middleware) {...})`.

```go
var RequestIdMiddleware = gonest.NewMiddleware(func(middleware *gonest.Middleware) {
  middleware.Handler(func(req *gonest.Request, res *gonest.Response, next gonest.Next) {
    res.SetHeader("X-Request-Id", "...")
    next(req, res)
  })
})
```

Llamar a `next(req, res)` continúa la cadena. **Omitirlo cortocircuita**
la solicitud — nada de flujo posterior (downstream) se ejecuta.

## Global vs. ámbito de controller

```go
var AppModule = gonest.NewModule(func(module *gonest.Module) {
  module.Use(RequestIdMiddleware) // global — root module only
})

var UserController = gonest.NewController(func(controller *gonest.Controller) {
  controller.Use(SomeOtherMiddleware) // controller-scoped
})
```

El middleware global (`Module.Use`, solo en el Module raíz) siempre se ejecuta **antes** que
el middleware a nivel de Controller (`Controller.Use`).

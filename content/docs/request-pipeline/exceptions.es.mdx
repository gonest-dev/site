---
title: Exceptions y Recuperación de Panic
description: HttpException, exceptions integradas y el contrato de recuperación de panic
---

`gonest.HttpException` es el tipo base que incorpora (embeds) cada exception de gonest.

```go
gonest.NewHttpException().
  SetStatus(http.StatusBadRequest).
  SetName("FooError").
  SetMessage("lorem ipsum").
  SetDetails(details)
```

Cualquier tipo que incorpore `HttpException` satisface estructuralmente la interfaz `Exception`
— la capa de recuperación usa una aserción de tipo (type-assertion), nunca un
type-switch cerrado.

## Exceptions integradas (Built-in)

| Exception | Status |
|---|---|
| `NotFoundException` | 404 |
| `BadRequestException` | 400 |
| `ConflictException` | 409 |
| `UnauthorizedException` | 401 |
| `ForbiddenException` | 403 |

Cada una tiene un constructor `New*Exception(details any)`.

## Exceptions personalizadas

Incorpora `gonest.HttpException` exactamente como una integrada:

```go
type FooError struct{ gonest.HttpException }

func NewFooError(details any) *FooError {
  return &FooError{
    HttpException: gonest.NewHttpException().
      SetStatus(http.StatusBadRequest).
      SetName("FooError").
      SetMessage("lorem ipsum").
      SetDetails(details),
  }
}
```

## Contrato de Panic → respuesta

<Callout type="info">
  Cualquier panic con un valor que satisfaga `Exception` se convierte en un JSON
  `{"name": ..., "message": ..., "details": ...}` en el propio código de estado de esa
  exception. Cualquier **otro** panic (puntero nulo, índice fuera de rango, etc.)
  se convierte en un error 500 genérico con **cero detalles internos filtrados** — el proceso
  nunca se bloquea (crashes).
</Callout>

## Constantes de estado HTTP

`gonest.HttpStatus*` reexporta las 63 constantes estándar `net/http.StatusXxx`
bajo el espacio de nombres `gonest` (ej: `gonest.HttpStatusNotFound`), por lo que
el código del handler nunca necesita importar `net/http` solo por los códigos de estado.

`gonest.NewProvider(func(provider *gonest.Provider) {...})`

| Método | Descrição |
|---|---|
| `Scope(gonest.ScopeSingleton \| ScopeTransient \| ScopeRequest)` | Define o tempo de vida (lifetime) da instância. |
| `Constructor(fn)` | `fn` é um de `func() T`, `func() (T, error)`, `func(ctx) T`, `func(ctx) (T, error)`. |

## Funções de resolução

| Função | Comportamento |
|---|---|
| `MustInject[T](owner)` | Resolve exatamente uma correspondência; dá panic se houver 0 ou N. |
| `MustInjectAll[T](owner)` | Resolve todas as correspondências; nunca dá panic. |
| `ProviderAs[T any](ref ProviderRef) ProviderRef` | Embrulha (wrap) `ref`, registrando-o como a interface `T` — o único jeito de `T` se tornar resolvível por `MustInject`/`MustInjectAll`. O registro concreto do próprio `ref` também precisa ser passado pra `Providers(...)`. Dá panic imediatamente se `T` não for uma interface, ou se `ref` for o resultado de outra chamada de `ProviderAs` (encadeamento). |

Veja: [Providers & DI Scopes](/docs/core-concepts/providers), [Dependency Injection](/docs/core-concepts/dependency-injection)

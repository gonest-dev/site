O `gonest.Emitter` é um **singleton do framework** — sempre resolvível via
`MustInject[*gonest.Emitter]` em qualquer módulo, com zero registro explícito.

```go
type UserCreatedEvent struct{ UserID int64 }

var UserCreatedListener = gonest.NewListener(func(listener *gonest.Listener[UserCreatedEvent]) {
  listener.MustOn(func(ctx context.Context, event UserCreatedEvent) {
    // handle the event
  })
})

var UserModule = gonest.NewModule(func(module *gonest.Module) {
  module.Listeners(UserCreatedListener)
})

var UserProvider = gonest.NewProvider(func(provider *gonest.Provider) {
  emitter := gonest.MustInject[*gonest.Emitter](provider)
  provider.Constructor(func() *UserService { return &UserService{emitter: emitter} })
})
```

<Callout type="info">
  `Listener[EventType]` é um tipo genérico de verdade — o argumento de tipo de `NewListener` é
  inferido a partir do próprio parâmetro da função builder. O builder recebe o `*Listener[EventType]`
  (pra `MustInject`/`MustInjectAll`) e deve chamar `On` ou `MustOn` exatamente uma vez: qualquer
  dependência é resolvida UMA VEZ, em tempo de bootstrap, e capturada no closure pra todo evento
  futuro.
</Callout>

<Callout type="info">
  `On(func(ctx, event) error)` e `MustOn(func(ctx, event))` diferem só em se o handler reporta falha
  — o erro retornado por `On` recebe o MESMO tratamento que `Emit` já dá a um panic recuperado
  (logado, nunca propagado); `MustOn` embrulha um handler que nunca falha, pra não precisar escrever
  `return nil`. Apesar do nome, `MustOn` NÃO panica em nada — exceção deliberada à convenção usual
  deste framework de que prefixo `Must` significa "panica", escolhida pra ler natural ao lado de `On`.
</Callout>

## Semântica do Emit

`Emitter.Emit(event)` é **fire-and-forget** (dispare e esqueça): uma goroutine por ouvinte (listener) registrado
para aquele tipo exato de evento, retorna imediatamente, nunca bloqueia quem
chamou (caller). O panic ou erro de um ouvinte é recuperado internamente e nunca
se propaga.

**Não há correspondência de eventos por padrão de string/wildcard** — os eventos são tipados por
struct do Go.

## Subscriptions dinâmicas (`Subscribe[T]`)

`NewListener`+`On`/`MustOn` registra um handler estático, com a vida toda do app. `gonest.Subscribe[T](emitter, done)`
é seu par dinâmico — um canal que vive só enquanto `done` continua aberto (tipicamente a vida de uma
conexão), fechado automaticamente assim que dispara. É sobre isso que as Subscriptions de GraphQL
são construídas (ver [GraphQL](/docs/graphql)):

```go
func(ctx *gonest.GraphqlContext, emit func(any)) {
  for event := range gonest.Subscribe[OrderCreatedEvent](emitter, ctx.Done()) {
    emit(event)
  }
}
```

---
title: Event Emitter
description: Typed, fire-and-forget events between providers
---

`gonest.Emitter` es un **singleton del framework** — siempre resoluble a través de
`MustInject[*gonest.Emitter]` en cualquier módulo, sin ningún registro explícito.

```go
type UserCreatedEvent struct{ UserID int64 }

var UserCreatedListener = gonest.NewListener(func(listener *gonest.Listener) {
  gonest.MustOn[UserCreatedEvent](listener, func(ctx context.Context, event UserCreatedEvent) {
    // handle the event
  })
})

var UserModule = gonest.NewModule(func(module *gonest.Module) {
  module.Listeners(UserCreatedListener)
})

var UserProvider = gonest.NewProvider(func(provider *gonest.Provider) {
  emitter := gonest.MustInject[*gonest.Emitter](provider)
  provider.Constructor(func() *UserService { return &UserService{emitter: emitter} })
})
```

<Callout type="info">
  `MustOn[EventType](listener, handler)` es una **función** libre, no un
  método — Go no permite parámetros de tipo (type parameters) en los métodos.
</Callout>

## Semántica de Emit

`Emitter.Emit(event)` es **fire-and-forget** (dispara y olvida): una goroutine por oyente (listener) registrado
para ese tipo exacto de evento, regresa inmediatamente, nunca bloquea al que
llama (caller). El panic o error de un oyente se recupera internamente y nunca
se propaga.

**No hay coincidencia de eventos por patrón de cadena/wildcard** — los eventos están tipados por
struct de Go.

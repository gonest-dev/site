title: OpenAPI & Swagger
description: Gerando documentação OpenAPI a partir dos mesmos schemas que validam
---

O mesmo `*Schema` que valida uma requisição também gera o OpenAPI —
nunca há uma segunda declaração paralela.

```go
doc := gonest.NewOpenAPI("3.1.0", func(b *gonest.OpenAPI) {
  b.Title("Example API")
  b.Version("1.0.0")
  b.BearerAuth()
})

gonest.OpenapiGenerate(app, doc) // percorre toda a árvore de módulos

gonest.SetupSwagger(app, "/docs", doc, gonest.SwaggerOptions{
  JsonDocumentUrl: "/openapi.json",
  PersistAuth:     true,
})
```

## Construtor `OpenAPI`

`Title`/`Description`/`Version`/`Contact(name, url, email)`/`License(name, url)`/`BearerAuth()`,
cada um com um getter correspondente (`TitleText()`, `ContactInfo()`, `HasBearerAuth()`, etc.).

## Métodos de documentação de rotas

| Método | Descrição |
|---|---|
| `Summary(...)` | Resumo curto da operação. |
| `Description(...)` | Descrição mais longa da operação. |
| `OperationId(...)` | `operationId` do OpenAPI. |
| `Tags(...)` | Sobrescreve as tags do Controller para esta rota. |
| `BearerAuth()` | Sobrescreve a configuração bearer-auth do Controller para esta rota. |
| `RequestBody(schema)` | Documenta o schema do corpo da requisição. |
| `Response(status, schema...)` | Documenta uma resposta; variádico — 0 argumentos documenta um status sem corpo; repita para múltiplos status (o mesmo status sobrescreve). |
| `PathParams(schema)` | Documenta os parâmetros de rota (path). |
| `QueryParams(schema)` | Documenta os parâmetros de query. |
| `ExcludeFromDocs()` | Oculta esta rota das documentações geradas. |
| `Deprecated()` | Marca esta rota como descontinuada (deprecated). |

## Geração de Schema

`OpenapiGenerate` percorre toda a árvore de módulos (raiz + importados,
recursivamente), deduplica referências de `*Schema` repetidas **por identidade de ponteiro**
(emitindo reuso de `$ref` em `components.schemas`), e cobre todas as famílias de ramificação
(String/Numeric/Boolean/DateTime/Array/Object).

<Callout type="warn">
  Campos [`Custom(fn)`](/docs/validation/custom-validators) aparecem nos
  schemas gerados com nome/required/nullable/description, mas **sem
  type/format** — uma closure não tem formato declarativo para deduzir. Essa é uma
  limitação conhecida e permanente, não um bug a ser corrigido.
</Callout>

## Swagger UI

`SetupSwagger` registra 2 rotas simples diretamente no adaptador — sem
envolvimento de DI/Module — uma servindo o JSON `doc.Document()`, outra servindo uma
página HTML carregando a Swagger UI via CDN (sem recursos/vendored localmente). Configure via
`SwaggerOptions{JsonDocumentUrl, PersistAuth, DocExpansion}`.

---
title: OpenAPI y Swagger
description: Generando documentación OpenAPI a partir de los mismos schemas que validan
---

El mismo `*Schema` que valida una solicitud también genera OpenAPI —
nunca una segunda declaración paralela.

```go
doc := gonest.NewOpenAPI("3.1.0", func(b *gonest.OpenAPI) {
  b.Title("Example API")
  b.Version("1.0.0")
  b.BearerAuth()
})

gonest.OpenapiGenerate(app, doc) // recorre todo el árbol de módulos

gonest.SetupSwagger(app, "/docs", doc, gonest.SwaggerOptions{
  JsonDocumentUrl: "/openapi.json",
  PersistAuth:     true,
})
```

## Constructor `OpenAPI`

`Title`/`Description`/`Version`/`Contact(name, url, email)`/`License(name, url)`/`BearerAuth()`,
cada uno con un getter correspondiente (`TitleText()`, `ContactInfo()`, `HasBearerAuth()`, etc.).

## Métodos de documentación de rutas

| Método | Descripción |
|---|---|
| `Summary(...)` | Resumen corto de la operación. |
| `Description(...)` | Descripción más larga de la operación. |
| `OperationId(...)` | `operationId` de OpenAPI. |
| `Tags(...)` | Sobrescribe las etiquetas del Controller para esta ruta. |
| `BearerAuth()` | Sobrescribe la configuración bearer-auth del Controller para esta ruta. |
| `RequestBody(schema)` | Documenta el schema del cuerpo de la solicitud. |
| `Response(status, schema...)` | Documenta una respuesta; variádico — 0 argumentos documenta un estado sin cuerpo; repite para múltiples estados (el mismo estado sobrescribe). |
| `PathParams(schema)` | Documenta los parámetros de ruta (path). |
| `QueryParams(schema)` | Documenta los parámetros de query. |
| `ExcludeFromDocs()` | Oculta esta ruta de las documentaciones generadas. |
| `Deprecated()` | Marca esta ruta como obsoleta (deprecated). |

## Generación de Schema

`OpenapiGenerate` recorre todo el árbol de módulos (raíz + importados,
recursivamente), deduplica las referencias repetidas de `*Schema` **por identidad de puntero**
(emitiendo reutilización de `$ref` en `components.schemas`), y cubre cada familia de ramas
(String/Numeric/Boolean/DateTime/Array/Object).

<Callout type="warn">
  Los campos [`Custom(fn)`](/docs/validation/custom-validators) aparecen en
  los schemas generados con name/required/nullable/description, pero **sin
  type/format** — un closure no tiene forma declarativa de la cual inferir. Esta es una
  limitación conocida y permanente, no un error a corregir.
</Callout>

## Swagger UI

`SetupSwagger` registra 2 rutas simples directamente en el adaptador — sin
participación de DI/Module — una sirviendo el JSON `doc.Document()`, otra sirviendo una
página HTML que carga Swagger UI a través de CDN (sin recursos locales/vendored). Configura a través de
`SwaggerOptions{JsonDocumentUrl, PersistAuth, DocExpansion}`.

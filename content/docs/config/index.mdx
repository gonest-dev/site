title: Config Loading
description: Load .env files with full dotenvx parity, then bind and validate typed config structs from env vars, reusing the same Schema/Parse[T] REST already uses
---

`gonest.Dotenv()` is a singleton â€” no DI, callable from `main()` before any `Module`/`Provider`/`NewApp`
exists. It plays two roles: loading a `.env` file into the process, and (once loaded) acting as a
`Parseable` source for `gonest.MustParse[T]`, exactly like any REST source.

```go
package main

import "gonest.dev/gonest"

func main() {
  gonest.Dotenv().MustLoad("./.env")

  app, err := gonest.NewApp[gonest.FiberApp](AppModule)
  if err != nil {
    panic(err)
  }

  err = app.Listen(":3000")
  if err != nil {
    panic(err)
  }
}
```

<Callout type="info">
  `Load(paths ...string)`/`MustLoad(paths ...string)` accept multiple paths â€” **first path wins**: a
  variable already set by an earlier path (or already present in the process environment before
  `Load` runs) is never overwritten by a later one, mirroring `dotenvx run -f`'s own precedence.
</Callout>

## `.env` syntax â€” full `dotenvx` parity

The parser matches [`dotenvx`](https://dotenvx.com/docs/env-file)'s real behavior, not a minimal
subset:

```bash
# full-line comment
DB_HOST=db.internal          # inline comment
DB_NAME='literal $VAR'       # single quotes: no interpolation, no escapes
DB_URL="postgres://${DB_HOST}/app"   # double quotes: interpolation + escapes

# 4 default/alternate operators, same as POSIX shell parameter expansion
DB_PORT=${PORT:-5432}        # use default if PORT unset OR empty
DB_USER=${USER-admin}        # use default only if USER unset
DB_LABEL=${FLAG:+enabled}    # use alternate if FLAG set AND non-empty
DB_FLAG=${FLAG+yes}          # use alternate only if FLAG is set

PRIVATE_KEY=`-----BEGIN KEY-----
multi
line
-----END KEY-----`
```

Escapes (`\n`/`\r`/`\t`/`\\`) apply inside double quotes only, matching `dotenvx` exactly.

## Env â†’ Schema Binding

`*Dotenv` satisfies `execution.Parseable` via `ParseInto` â€” `gonest.MustParse[T](gonest.Dotenv(), schema)`
works identically to any REST `Parse[T]` call, reusing the same validation pipeline (`resolveSchema`,
`coerceParamString`, `validateValue`, `populate`) unchanged:

```go
type DatabaseConfig struct {
  Host string `json:"host" env:"DB_HOST"`
  Port int64  `json:"port" env:"DB_PORT"`
  Url  string `json:"url"  env:"DB_URL"`
}

var databaseConfigSchema = gonest.NewSchema(func(t *DatabaseConfig, s *gonest.Schema) {
  s.Property(&t.Host).String().Required()
  s.Property(&t.Port).Integer().Default(int64(5432))
  s.Property(&t.Url).String().Required()
})

var DatabaseConfigProvider = gonest.NewProvider(func(provider *gonest.Provider) {
  provider.Constructor(func() *DatabaseConfig {
    return gonest.MustParse[*DatabaseConfig](gonest.Dotenv(), databaseConfigSchema)
  })
})
```

<Callout type="info">
  `env:"NAME"` maps a field to its env var name, the same pattern as `param`/`query` tags.
  `PropertyBuilder.Default(value)` supplies a fallback when the source (currently `env` only) has no
  value for that field â€” the field no longer triggers `Required` in that case.
</Callout>

<Callout type="info">
  `T` in `MustParse[T]`/`Parse[T]` can be the struct itself or a pointer to it. `MustParse[*DatabaseConfig]`
  allocates the struct and hands back the populated pointer directly â€” exactly what a `Provider.Constructor`
  returning `*DatabaseConfig` needs, no manual `config := MustParse[DatabaseConfig](...); return &config`.
</Callout>

<Callout type="warn">
  `Schema.Validate(instance)` (validating an already-built struct) does **not** exist and was
  deliberately rejected â€” it would open a second, parallel validation path in `Schema`. `Parse[T]`
  always validates against a raw source, never a populated instance.
</Callout>

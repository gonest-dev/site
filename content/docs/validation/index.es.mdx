description: NewSchema, campos identificados por puntero y las restricciones base
---

Toda funcionalidad de validación y OpenAPI de gonest se basa en un fundamento:
`gonest.NewSchema[T](func(t *T, s *gonest.Schema) {...})`.

```go
type UserEntity struct {
  Id    int64  `json:"id"`
  Name  string `json:"name"`
  Email string `json:"email"`
}

var userEntitySchema = gonest.NewSchema[UserEntity](func(t *UserEntity, s *gonest.Schema) {
  s.Title("UserEntity")
  s.Property(&t.Id).Integer().Required()
  s.Property(&t.Name).String().Required().Min(1).Max(50)
  s.Property(&t.Email).Email().Required()
})
```

<Callout type="info">
  Los campos se identifican por su propia **dirección de puntero**
  (`m.Property(&t.Name)`) — no una struct tag, ni un nombre de campo de cadena. Renombra
  el campo y el compilador capta todas las referencias; no hay una cadena (string) que
  se desincronice.
</Callout>

## Restricciones base

Cada rama (`String()`, `Integer()`, `Array()`, `Object()`, ...) devuelve un
constructor encadenable (chainable builder) que siempre tiene estos cuatro:

| Método | Efecto |
|---|---|
| `Required()` | El campo debe estar presente (y no nulo a menos que `Nullable()`). |
| `Nullable()` | Acepta `null` de JSON incluso cuando está `Required()`. |
| `Description("...")` | Descripción del campo OpenAPI. |
| `Examples(...)` | Valores de ejemplo OpenAPI. |

`m.Title("...")` establece el nombre bajo el cual aparece este schema en
`components.schemas` de OpenAPI — por defecto es el nombre del tipo de Go.

<Callout type="warn">
  Nota histórica: los documentos de diseño de `.specs` anteriores usaban `NewMetadata[T]` /
  `*Metadata` y `MustJsonBody[T]`, luego una revisión posterior previa al lanzamiento usó
  una familia de funciones de análisis prefijadas con `Must...` específicas de body/params/query.
  Todas estas fueron reemplazadas antes del lanzamiento — `NewSchema[T]` / `*Schema`
  y `gonest.MustParse[T](src, schema)` (con `src` de
  `req.Body().Json()`, `req.Params()`, `req.Query()`, `req.Headers()`, o
  `req.Body().Form(onFile)`) son los nombres correctos actuales. Este sitio utiliza
  solo la API actual.
</Callout>

Três famílias de tags separadas mapeiam para três fontes separadas: `param:"..."`,
`query:"..."`, e `json:"..."`.

```go
type UserIdParams struct {
  UserId int64 `param:"user_id"`
}

var userIdParamsSchema = gonest.NewSchema[UserIdParams](func(t *UserIdParams, s *gonest.Schema) {
  s.Property(&t.UserId).Integer().Min(1).Required()
})

var UserController = gonest.NewController(func(controller *gonest.Controller) {
  controller.RouteGet("/:user_id", func(route *gonest.Route) {
    route.Params(userIdParamsSchema)
    route.Response(gonest.HttpStatusOk, func(response *gonest.Response) {
      response.Schema(userEntitySchema)
    })
    route.Handler(func(req *gonest.Request, res *gonest.Response) {
      params := gonest.MustParse[UserIdParams](req.Params(), userIdParamsSchema)
      res.Json(params)
    })
  })
})
```

## Funções de parsing (Parsing functions)

A validação sempre passa por um par unificado:

- `gonest.Parse[T](src, schema) (T, error)` — sem panic.
- `gonest.MustParse[T](src, schema) T` — dá panic em falha de validação (o
  panic é transformado em uma resposta `BadRequestException` pelo framework).

`T` é o tipo struct simples (ex: `UserIdParams`), não um ponteiro.

`src` é um `Parseable` obtido a partir de uma das cinco fontes em `*gonest.Request`:

| Fonte (Source) | Lê de |
|---|---|
| `req.Params()` | Parâmetros de rota (tags `param:"..."`) |
| `req.Query()` | Query string (tags `query:"..."`) |
| `req.Headers()` | Cabeçalhos HTTP |
| `req.Body().Json()` | Corpo JSON da requisição (tags `json:"..."`) |
| `req.Body().Form(onFile)` | Corpo de formulário multipart (veja [Multipart](/docs/multipart)) |

## Comportamento da validação

- **Toda** violação é coletada — a validação nunca falha rapidamente (fails fast) no
  primeiro erro.
- A validação é **recursiva**: itens de array são checados individualmente, e
  `$ref`s de objetos são adentrados.
- Os `details` da `BadRequestException` contêm uma lista ordenada de `{field, message}`.

<Callout type="warn">
  `Nullable()` + um valor `null` no JSON é aceito mesmo em um campo `Required`.
  Mas um campo `Required` cuja chave está **totalmente ausente** no payload é
  sempre uma violação — **ausência não é a mesma coisa que nulo**.
</Callout>

## Atualizações parciais (`Accessor[T]`)

`gonest.Accessor[T]` rastreia se um campo estava explicitamente presente no corpo
JSON — incluindo `null` explícito — para handlers no estilo PATCH que
devem alterar apenas o que o cliente realmente enviou:

```go
type UpdateUserDTO struct {
  Name  gonest.Accessor[string] `json:"name"`
  Email gonest.Accessor[string] `json:"email"`
}

body := gonest.MustParse[*UpdateUserDTO](req.Body().Json(), updateUserSchema)
body.Name.OnDirty(func(name string) { user.Name = name })
body.Email.Apply(&user.Email)

// Sync escreve em outro Accessor[T] em vez de um *T cru -- o estado dirty
// propaga junto com o valor (ex: DTO -> entidade, ambos Accessor)
body.Email.Sync(&entity.Email)

// Sincroniza automaticamente todos os campos Accessor em estado dirty do body para a entidade:
gonest.SyncAccessorFields(entity, body)
```

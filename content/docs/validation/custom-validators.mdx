---
title: Custom Validators
description: The Custom(fn) escape hatch
---

`Custom(fn func(raw any) (any, error))` is a universal escape hatch available
on every `PropertyBuilder` — it works identically across JSON body, path
params, query string, and form fields.

```go
m.Property(&t.Slug).Custom(func(raw any) (any, error) {
  s, ok := raw.(string)
  if !ok || !slugPattern.MatchString(s) {
    return nil, fmt.Errorf("must be a URL-safe slug")
  }
  return s, nil
})
```

`fn`'s returned error becomes a violation on that field. Its returned value
populates the struct field.

Use `Custom` for domain-specific formats or transforms the fixed vocabulary
(String/Numeric/Boolean/DateTime/Array/Object) can't express.

<Callout type="info">
  This absorbed the intent of an earlier, now-removed pipeline stage called
  `Pipe`/`NewPipe`. If you've seen `Pipe` mentioned in older gonest design
  docs, `Custom(fn)` is its replacement — `Pipe` no longer exists in the
  current API.
</Callout>

<Callout type="warn">
  Known limitation: `Custom(fn)` fields appear in generated OpenAPI schemas
  with their name/required/nullable/description, but **no type or format** —
  a closure has no declarative shape to infer from. See
  [OpenAPI & Swagger](/docs/openapi).
</Callout>

## Sanitize: pre-processing that composes

`Custom(fn)` **replaces** every built-in check (Min/Max/Pattern never run
afterwards). `Sanitize(fn func(raw any) any)` is different: it transforms
`raw` **before** anything else runs, but the built-in checks (or `Custom`,
if both are set) still run afterwards, on the transformed value:

```go
m.Property(&t.Cpf).String().Min(11).Max(11).Pattern(`^\d{11}$`).Sanitize(func(raw any) any {
  s, _ := raw.(string)
  return strings.TrimSpace(s) // "  12345678901  " still passes Min/Max/Pattern
}).Required()
```

`Sanitize` must come **after** `Min`/`Max`/`Pattern` in the chain (it
returns a bare `*PropertyBuilder`, same as `Custom` — those two live on the
branch-specific wrapper, e.g. `*StringSchema`).

<Callout type="info">
  `fn` may run twice per request (once during validation, once during
  population) — it must be idempotent, same contract as `Custom(fn)`.
</Callout>

## Refine: cross-field checks

Some checks need more than one field — e.g. a password confirmation.
`Schema.Refine(fn func(dst any) (field string, err error))` registers a
check that runs only **after** every individual field has validated and
been populated:

```go
var updateUserSchema = gonest.NewSchema[UpdateUserDTO](func(t *UpdateUserDTO, s *gonest.Schema) {
  s.Property(&t.Password).String().Min(8).Required()
  s.Property(&t.ConfirmPassword).String().Min(8).Required()

  s.Refine(func(dst any) (field string, err error) {
    d := dst.(*UpdateUserDTO)
    if d.Password != d.ConfirmPassword {
      return "confirmPassword", errors.New("must match password")
    }
    return "", nil
  })
})
```

Calling `Refine` more than once accumulates checks — every one runs
(never stops at the first failure), each contributing at most one
violation.

<Callout type="warn">
  V1 scope: `Refine` only runs for JSON body (`req.Body().Json()`).
  Params/query/form/headers aren't covered yet.
</Callout>

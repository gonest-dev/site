Toda funcionalidade de validação e OpenAPI do gonest é construída sobre uma base:
`gonest.NewSchema[T](func(t *T, m *gonest.Schema) {...})`.

```go
type UserEntity struct {
  Id    int64  `json:"id"`
  Name  string `json:"name"`
  Email string `json:"email"`
}

var userEntitySchema = gonest.NewSchema[UserEntity](func(t *UserEntity, m *gonest.Schema) {
  m.Title("UserEntity")
  m.Property(&t.Id).Integer().Required()
  m.Property(&t.Name).String().Required().Min(1).Max(50)
  m.Property(&t.Email).Email().Required()
})
```

<Callout type="info">
  Os campos são identificados pelo seu próprio **endereço de ponteiro**
  (`m.Property(&t.Name)`) — não uma struct tag, não um nome de campo em string. Renomeie
  o campo e o compilador pega todas as referências; não há string para
  ficar fora de sincronia.
</Callout>

## Restrições base

Toda ramificação (`String()`, `Integer()`, `Array()`, `Object()`, ...) retorna um
builder encadeável (chainable builder) que sempre tem estes quatro:

| Método | Efeito |
|---|---|
| `Required()` | O campo deve estar presente (e não-nulo a menos que tenha `Nullable()`). |
| `Nullable()` | Aceita o `null` do JSON mesmo com `Required()`. |
| `Description("...")` | Descrição do campo OpenAPI. |
| `Examples(...)` | Valores de exemplo OpenAPI. |

`m.Title("...")` define o nome com o qual este schema aparece no
`components.schemas` do OpenAPI — o padrão é o nome do tipo Go.

<Callout type="warn">
  Nota histórica: documentos de design antigos em `.specs` usavam `NewMetadata[T]` /
  `*Metadata` e `MustJsonBody[T]`, em seguida, uma revisão de pré-lançamento posterior usou
  uma família de funções de parsing prefixadas com `Must...` específicas para body/params/query.
  Todas estas foram substituídas antes do lançamento — `NewSchema[T]` / `*Schema`
  e `gonest.MustParse[T](src, schema)` (com `src` sendo
  `req.Body().Json()`, `req.Params()`, `req.Query()`, `req.Headers()`, ou
  `req.Body().Form(onFile)`) são os nomes corretos atuais. Este site usa
  apenas a API atual.
</Callout>

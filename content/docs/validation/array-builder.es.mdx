---
title: Array Builder
description: El patrón de doble estado container/item
---

`Property(&t.Tags).Array()` devuelve un `*ArrayMetadata` — un constructor
**de doble estado** (dual-state). Configura el campo contenedor o el elemento, dependiendo de
dónde llames a las funciones.

<Callout type="warn">
  Este es el punto más común de confusión en la API de schema. `.Items(func(m
  *gonest.ArrayMetadata) {...})` es un **callback**, no una lista variádica.
  **Dentro** del callback, `m.String()`/`m.Integer()`/`m.Object(ref)` etc.
  configuran el **elemento (item)**. Llamar a `m.Required()`/`m.Nullable()`/`m.Description()`/`m.Examples()`
  **fuera** del callback (directamente en el retorno de `.Array()`) siempre
  configura el **campo contenedor**, nunca el elemento.
</Callout>

```go
m.Property(&t.Tags).Array().Items(func(m *gonest.ArrayMetadata) {
  m.String().Min(1).Max(50) // item: each string 1-50 chars
  m.Required()              // item: required (non-null entries)
  m.Description("Tags do usuário")
  m.Examples("admin", "beta")
})
```

## Límites de cantidad de elementos vs. límites del elemento

`Items(fn)` devuelve los metadatos del array en sí, por lo que encadenar `.Min(n)`/`.Max(n)`
**después** de `Items(fn)` establece los límites en el **número de elementos** en el array —
distinto de los propios `Min`/`Max` del elemento establecidos dentro del callback.

```go
m.Property(&t.Tags).Array().Items(func(m *gonest.ArrayMetadata) {
  m.String().Min(1).Max(50) // each item: 1-50 characters
}).Min(1).Max(10) // the array itself: 1-10 items
```

## Arrays de objetos anidados (reutilización de $ref)

```go
m.Property(&t.Addresses).Array().Items(func(m *gonest.ArrayMetadata) {
  m.Object(addressMetadata) // reuse an already-registered schema
  m.Required()
}).Min(1)
```

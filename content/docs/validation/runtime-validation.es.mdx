Tres familias de etiquetas separadas se asignan a tres fuentes separadas: `param:"..."`,
`query:"..."`, y `json:"..."`.

```go
type UserIdParams struct {
  UserId int64 `param:"user_id"`
}

var userIdParamsSchema = gonest.NewSchema[UserIdParams](func(t *UserIdParams, s *gonest.Schema) {
  s.Property(&t.UserId).Integer().Min(1).Required()
})

var UserController = gonest.NewController(func(controller *gonest.Controller) {
  controller.RouteGet("/:user_id", func(route *gonest.Route) {
    route.Params(userIdParamsSchema)
    route.Response(gonest.HttpStatusOk, func(response *gonest.Response) {
      response.Schema(userEntitySchema)
    })
    route.Handler(func(req *gonest.Request, res *gonest.Response) {
      params := gonest.MustParse[UserIdParams](req.Params(), userIdParamsSchema)
      res.Json(params)
    })
  })
})
```

## Funciones de parsing (Parsing functions)

La validación siempre pasa por un par unificado:

- `gonest.Parse[T](src, schema) (T, error)` — sin panic.
- `gonest.MustParse[T](src, schema) T` — da panic en falla de validación (el
  panic se convierte en una respuesta `BadRequestException` por el framework).

`T` es el tipo de struct simple (ej: `UserIdParams`), no un puntero.

`src` es un `Parseable` obtenido de una de cinco fuentes en `*gonest.Request`:

| Fuente (Source) | Lee de |
|---|---|
| `req.Params()` | Parámetros de ruta (etiquetas `param:"..."`) |
| `req.Query()` | Query string (etiquetas `query:"..."`) |
| `req.Headers()` | Encabezados HTTP |
| `req.Body().Json()` | Cuerpo JSON de la solicitud (etiquetas `json:"..."`) |
| `req.Body().Form(onFile)` | Cuerpo de formulario multipart (consulta [Multipart](/docs/multipart)) |

## Comportamiento de validación

- **Cada** violación es recolectada — la validación nunca falla rápidamente (fails fast) en el
  primer error.
- La validación es **recursiva**: los elementos del array se comprueban individualmente, y
  se recorren los `$ref`s de los objetos.
- Los `details` de `BadRequestException` contienen una lista ordenada de `{field, message}`.

<Callout type="warn">
  `Nullable()` + un valor `null` JSON es aceptado incluso en un campo `Required`.
  Pero un campo `Required` cuya clave está **completamente ausente** del payload es
  siempre una violación — **la ausencia no es lo mismo que nulo**.
</Callout>

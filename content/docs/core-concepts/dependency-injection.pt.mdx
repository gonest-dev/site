title: Injeção de Dependência
description: MustInject, MustInjectAll e multi-binding

O gonest resolve dependências pelo **tipo**, não por nome ou token de string.

## `MustInject[T](owner)`

Resolve exatamente uma instância de `T`. Dá panic se zero ou mais de uma correspondência for
encontrada.

- **Tipos de ponteiro** (`*UserService`) exigem uma correspondência exata de tipo.
- **Tipos de interface** correspondem a um registro exato, ou utilizam
  `reflect.Type.Implements()` como alternativa se exatamente um Provider satisfizer a interface.

```go
userService := gonest.MustInject[*UserService](controller)
```

## `MustInjectAll[T](owner)`

Resolve **todos** os Providers que correspondem a `T`. Nunca dá panic em 0 ou N correspondências — use-o
para padrões de plugin/estratégia (plugin/strategy patterns) onde vários Providers validamente
implementam a mesma interface.

```go
type Pingable interface {
  Name() string
  Ping(ctx context.Context) error
}

var HealthController = gonest.NewController(func(controller *gonest.Controller) {
  controller.Path("/health")
  pingables := gonest.MustInjectAll[Pingable](controller)
  _ = pingables
})
```

<Callout type="info">
  Esse é o mesmo padrão usado para [Health Checks](/docs/health-checks): qualquer
  número de serviços pode implementar `Pingable`, e o Controller de saúde descobre todos eles automaticamente.
</Callout>

## Onde `MustInject` pode ser chamado

`MustInject`/`MustInjectAll` são legais dentro de construtores de `Provider`, `Controller`
— em qualquer lugar que o framework garanta que a Fase 1 (resolução de Providers) já
foi concluída. Veja [Bootstrap](/docs/getting-started/bootstrap).

Os construtores de `Guard` são a única exceção: eles **não** têm suporte a `MustInject`,
porque um único Guard pode ser anexado em vários Controllers/Modules —
não há um único "proprietário" (owner) para resolver em relação a ele.

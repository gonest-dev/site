---
title: Módulos
description: Conectando providers, controllers y etapas del pipeline
---

Un `Module` es la unidad que gonest inicializa. Construye uno con
`gonest.NewModule(func(module *gonest.Module) {...})`.

```go
var UserModule = gonest.NewModule(func(module *gonest.Module) {
  module.Providers(UserProvider)
  module.Controllers(UserController)
})
```

## Métodos

| Método | Descripción |
|---|---|
| `Providers(...)` | Registra Providers propiedad de este Module. |
| `Controllers(...)` | Registra Controllers propiedad de este Module. |
| `Imports(...)` | Importa otros Modules, obteniendo acceso a sus Providers exportados. |
| `Exports(...)` | Expone Providers a Modules que importan este. |
| `Use(...)` | Registra Middleware global. **Solo en el Module raíz.** |
| `Filters(...)` | Registra Filters de excepción globales. **Solo en el Module raíz.** |
| `Listeners(...)` | Registra escuchas de eventos para el [Event Emitter](/docs/emitter). |
| `Schedulers(...)` | Registra trabajos del [Scheduler](/docs/scheduler). |
| `Name("...")` | Establece un nombre legible utilizado en mensajes de error. |

<Callout type="warn">
  `Use` y `Filters` son solo para la raíz cuando se aplican a nivel de Module — se
  aplican globalmente a todas las solicitudes. `Use`/`Filters` por Controller también existen
  (consulta [Request Pipeline](/docs/request-pipeline)) y se combinan con los globales.
</Callout>

## Aplicando Middleware y Filters globales

```go
var AppModule = gonest.NewModule(func(module *gonest.Module) {
  module.Use(RequestIdMiddleware)
  module.Filters(FooFilter)
})
```

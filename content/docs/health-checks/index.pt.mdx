---
title: Verificações de Saúde
description: Rotas de readiness e liveness no estilo Terminus
---

Não há um tipo de bootstrap dedicado para health checks — um health check é
deliberadamente apenas um `Controller` normal, espelhando como o próprio módulo Terminus do NestJS
funciona conceitualmente.

```go
type Pingable interface {
  Name() string
  Ping(ctx context.Context) error
}

var HealthController = gonest.NewController(func(controller *gonest.Controller) {
  controller.Path("/health")
  pingables := gonest.MustInjectAll[Pingable](controller)

  controller.RouteGet("/readyz", func(route *gonest.Route) {
    route.Handler(func(req *gonest.Request, res *gonest.Response) {
      status := gonest.HttpStatusOk
      for _, p := range pingables {
        if p.Ping(context.Background()) != nil {
          status = gonest.HttpStatusServiceUnavailable
        }
      }
      res.Status(status).Json(map[string]string{"status": "ok"})
    })
  })

  controller.RouteGet("/livez", func(route *gonest.Route) {
    route.Handler(func(req *gonest.Request, res *gonest.Response) {
      res.Status(gonest.HttpStatusOk).Text("OK")
    })
  })
})
```

- **`/readyz`** (readiness): faz ping em todo `Pingable` registrado via
  [`MustInjectAll`](/docs/core-concepts/dependency-injection) — 200 se todos estiverem
  operantes, 503 se algum estiver inoperante.
- **`/livez`** (liveness): um 200 estático via `res.Status(...).Text("OK")` —
  `Text` força `Content-Type: text/plain`, um corpo bruto e não-JSON provando que o
  processo não está em deadlock (travado).

`Response.Text(s string) error` e as constantes `gonest.HttpStatusOk` /
`gonest.HttpStatusServiceUnavailable` são primitivas de propósito geral, não específicas do Terminus.

---
title: Comprobaciones de Salud
description: Rutas de readiness y liveness al estilo Terminus
---

No hay un tipo de bootstrap dedicado para health checks — un health check es
deliberadamente solo un `Controller` normal, reflejando cómo funciona conceptualmente
el propio módulo Terminus de NestJS.

```go
type Pingable interface {
  Name() string
  Ping(ctx context.Context) error
}

var HealthController = gonest.NewController(func(controller *gonest.Controller) {
  controller.Path("/health")
  pingables := gonest.MustInjectAll[Pingable](controller)

  controller.RouteGet("/readyz", func(route *gonest.Route) {
    route.Handler(func(req *gonest.Request, res *gonest.Response) {
      status := gonest.HttpStatusOk
      for _, p := range pingables {
        if p.Ping(context.Background()) != nil {
          status = gonest.HttpStatusServiceUnavailable
        }
      }
      res.Status(status).Json(map[string]string{"status": "ok"})
    })
  })

  controller.RouteGet("/livez", func(route *gonest.Route) {
    route.Handler(func(req *gonest.Request, res *gonest.Response) {
      res.Status(gonest.HttpStatusOk).Text("OK")
    })
  })
})
```

- **`/readyz`** (readiness): hace ping a cada `Pingable` registrado a través de
  [`MustInjectAll`](/docs/core-concepts/dependency-injection) — 200 si todos están
  activos, 503 si alguno está caído.
- **`/livez`** (liveness): un 200 estático a través de `res.Status(...).Text("OK")` —
  `Text` fuerza `Content-Type: text/plain`, un cuerpo en bruto, no JSON, lo que demuestra que el
  proceso no está bloqueado (deadlocked).

`Response.Text(s string) error` y las constantes `gonest.HttpStatusOk` /
`gonest.HttpStatusServiceUnavailable` son primitivas de propósito general, no específicas de Terminus.

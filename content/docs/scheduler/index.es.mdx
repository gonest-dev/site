---
title: Scheduler
description: Trabajos de cron, interval y timeout
---

```go
var CleanupScheduler = gonest.NewScheduler(func(scheduler *gonest.Scheduler) {
  scheduler.Cron("cleanup", "0 0 * * *", func(ctx context.Context) {
    // runs every day at midnight
  })
  scheduler.Interval("ping", time.Minute, func(ctx context.Context) {
    // runs every minute
  })
  scheduler.Timeout("warmup", 5*time.Second, func(ctx context.Context) {
    // runs once, 5 seconds after bootstrap
  })
})

var AppModule = gonest.NewModule(func(module *gonest.Module) {
  module.Schedulers(CleanupScheduler)
})
```

- `Cron(name, expr, fn)` analiza una expresión cron estándar de 5 campos a través de
  `github.com/robfig/cron/v3`.
- `Interval(name, dur, fn)` se repite en cada intervalo (duration).
- `Timeout(name, dur, fn)` se dispara exactamente una vez.
- `Scheduler.Stop(name)` cancela un job en ejecución.

<Callout type="info">
  Cada ejecución de job está totalmente aislada — tiene su propio `recover`, su propia
  goroutine. Un panic en un job nunca descarrila el proceso ni bloquea la siguiente
  ejecución programada.
</Callout>

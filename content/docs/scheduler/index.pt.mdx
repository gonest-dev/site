---
title: Scheduler
description: Jobs de cron, interval e timeout
---

```go
var CleanupScheduler = gonest.NewScheduler(func(scheduler *gonest.Scheduler) {
  scheduler.Cron("cleanup", "0 0 * * *", func(ctx context.Context) {
    // runs every day at midnight
  })
  scheduler.Interval("ping", time.Minute, func(ctx context.Context) {
    // runs every minute
  })
  scheduler.Timeout("warmup", 5*time.Second, func(ctx context.Context) {
    // runs once, 5 seconds after bootstrap
  })
})

var AppModule = gonest.NewModule(func(module *gonest.Module) {
  module.Schedulers(CleanupScheduler)
})
```

- `Cron(name, expr, fn)` faz o parse de uma expressão cron padrão de 5 campos via
  `github.com/robfig/cron/v3`.
- `Interval(name, dur, fn)` repete a cada intervalo (duration).
- `Timeout(name, dur, fn)` dispara exatamente uma vez.
- `Scheduler.Stop(name)` cancela um job em execução.

<Callout type="info">
  Cada execução de job é totalmente isolada — tem seu próprio `recover`, sua própria
  goroutine. Um panic em um job nunca derruba o processo ou bloqueia a próxima
  execução agendada.
</Callout>
